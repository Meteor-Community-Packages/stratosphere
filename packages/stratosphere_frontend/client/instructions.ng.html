Stratosphere usage instructions

Installation

Follow the instructions in the README.

Usage

To publish and sync with Stratosphere, you have to set the package server URL to your stratosphere URL. This can be achieved by setting the environment variable: 

METEOR_PACKAGE_SERVER_URL
to your package server url.

IMPORTANT: You can not immediately publish stand-alone packages. Meteor first needs to create and fill up a new package database file. This is automatically done for you when you try to run an existing meteor project with the environment variable set. First time syncing can take long (up to 10min), since it needs to import all package data from the official package repo and do some processing on them.

Meteor splits the package database and package files according to this URL, so you should be able to switch package servers back and forth by changing this variable.

Access Control

Note that there is no way to protect who can see and synchronize packages! Therefore, you probably want to run Stratosphere inside a firewall protected part of your organization.
But you can protect who can publish and unpublish pacakges by setting 

public.loginRequired
in the settings.json file.
You will need to add at least one superUser in the settings.json file (a meteor developer account username). SuperUsers can manage other user rights in the frontend.
